# 模板语法详解

模板是 WikiPreprocessor 中一种**可复用的 HTML 片段**，允许用户通过简单的调用语法插入预定义的、带参数的 HTML 结构。

---

## 一、模板调用语法

### 基本格式

模板调用由**外层大括号**包裹，内部包含**模板名**和**参数**：

```
{{模板名}参数}
```

关键特征：
- 最外层是 `{...}`
- 内层先写 `{模板名}`，然后写参数
- 模板名后紧跟 `}` 闭合，参数在模板名闭合之后、外层闭合之前

### 单参数（无名）调用

当模板只有一个参数时，可以省略参数名：

```
{{好看的图表}172,163,105*144*97}
```

这会自动匹配模板中的**第一个插槽**。

### 多参数（有名）调用

```
{{名称信息}
中文名::充电宝
&& 英文名::Power Baby}
```

- 参数之间用 `&&` 分隔
- 参数名和参数值之间用 `::` 或 `：：` 分隔
- 参数名对应模板源码中的插槽名
- 支持换行，换行不影响解析

### 参数中的换行

```
{{重点强调} 文字 ::
不准写
敏感内容}
```

参数值可以包含换行，具体解析方式取决于插槽类型（纯文本 / 行内 / 块级）。

## 二、模板定义（插槽系统）

模板源码中通过**插槽标记**定义用户可以填写内容的位置。

### 四种插槽类型

| 插槽类型 | 源码写法 | 参数解析方式 | 用途 |
|----------|----------|-------------|------|
| **纯文本插槽** | `[__名称__]` | 不解析，原样插入 | 纯文本、HTML属性值 |
| **行内解析插槽** | `[[__名称__]]` | 仅行内解析 | 支持粗体、斜体、链接等 |
| **块解析插槽** | `[[[__名称__]]]` | 完整块级解析 | 支持标题、列表、表格等 |
| **唯一ID插槽** | `[__%名称%__]` | 自动生成唯一标识 | JS/CSS选择器、ID属性 |

### 插槽命名规则

- 支持中文、英文、数字、下划线
- 长度 1~10 个字符

## 三、完整示例

### 示例 1：信息卡片模板

**模板定义（源码）：**

```html
<div class="info-card">
  <h3>[__标题__]</h3>
  <div class="content">[[[__正文__]]]</div>
  <small>作者：[[__作者__]]</small>
</div>
```

**用户调用：**

```
{{信息卡片}
标题::产品介绍
&& 作者::张三
&& 正文::
# 功能特点
- 功能一
- 功能二}
```

**输出结果：**

```html
<div class="info-card">
  <h3>产品介绍</h3>
  <div class="content"><h1>功能特点</h1><div class="indent"><ul><li>功能一</li><li>功能二</li></ul></div></div>
  <small>作者：张三</small>
</div>
```

### 示例 2：带唯一ID的交互组件

**模板定义：**

```html
<div id="[__%组件ID%__]">
  <button onclick="toggle('[__%组件ID%__]')">切换</button>
  <div class="panel">[[[__内容__]]]</div>
</div>
```

**用户调用两次：**

```
{{折叠面板}内容::第一块内容}
{{折叠面板}内容::第二块内容}
```

**输出结果：**

```html
<div id="组件ID_0">
  <button onclick="toggle('组件ID_0')">切换</button>
  <div class="panel"><p>第一块内容</p></div>
</div>
<div id="组件ID_1">
  <button onclick="toggle('组件ID_1')">切换</button>
  <div class="panel"><p>第二块内容</p></div>
</div>
```

注意：同一个模板中的**所有同名唯一ID插槽**在同一次调用中获得**相同的值**，不同调用之间值递增。

### 示例 3：图表数据模板

**模板定义：**

```html
<div class="chart" data-values="[__数据__]"></div>
<script>
  initChart('[__数据__]');
</script>
```

**用户调用：**

```
{{图表}100,200,150,300}
```

**输出结果：**

```html
<div class="chart" data-values="100,200,150,300"></div>
<script>
  initChart('100,200,150,300');
</script>
```

## 四、与"植入"的区别

模板调用和植入（Implant）都使用大括号，但语法不同：

| 类型 | 语法 | 说明 |
|------|------|------|
| **模板调用** | `{{名称}参数}` | 双层大括号，有模板名和参数 |
| **植入** | `{内容}` | 单层大括号，直接交给 `IScopedConvertingProvider.Implant` 解析 |

植入示例：

```
{词条名}
```

如果 `Implant("词条名")` 返回 `<a href="/w/123">词条名</a>`，则输出该链接；否则保留原样 `{词条名}`。

## 五、模板定义（后端代码）

模板通过 `Template` 类定义，注册到 `ParserBuilder` 中：

```csharp
var template = new Template(
    name: "信息卡片",           // 模板名（用户调用时使用）
    source: "<div>...</div>",   // 模板源码（含插槽标记）
    styles: ".card{border:1px solid #ccc}",  // 可选：模板使用的CSS
    preScripts: "",             // 可选：前置JS（在内容前执行）
    postScripts: ""             // 可选：后置JS（在内容后执行）
);

var parser = new ParserBuilder()
    .Template.AddTemplates([template])
    .BuildParser();
```

### 安全提示

- 带 JavaScript 的模板（`preScripts` / `postScripts` 非空）**仅允许超级管理员创建**
- JS 的全局作用域只能声明 `var` 变量（预览刷新时会重复运行）
- 模板源码中的 HTML 标签不会被转义，需自行确保安全性

## 六、错误处理

| 错误情况 | 输出 |
|----------|------|
| 找不到模板 | `[[解析错误]]__找不到指定模板:(模板名)` |
| 未填写模板名 | `[[解析错误]]__模板调用格式：{{模板名}参数列表}，未填写模板名` |
| 参数格式错误 | `[[解析错误]]__参数列表格式：参数名A::参数值A && 参数名B::参数值B，...` |
| 大括号未闭合 | `[[解析错误]]__未闭合'{'与'}'` |
