AuTableFloat 表格浮动插件

在表格的任意单元格中写入触发词，即可让该表格浮动（形成文本环绕表格的效果），并自动移除包含触发词的那一行。

推荐搭配“隐藏标题”语法使用

触发词

  • au-table-float

基本用法

在表格任意单元格中填入触发词，可选后跟方向参数和移动端策略：

au-table-float
第一列数据第二列数据
注意：包含触发词的行会在页面渲染后被自动移除，用户看不到这一行。

方向参数

触发词后可跟 left 或 right 控制浮动方向，不区分大小写。不写时默认为 right。

au-table-float right向右浮动
au-table-float left向左浮动

移动端策略 mobile(...)

可指定在移动端（屏幕宽度小于阈值）时的处理策略，语法为 mobile(策略, 阈值, 按钮文本)。

  • 策略：可选 asis、enforce、drawer，默认 asis。
  • 阈值：像素值，默认 800px。屏幕宽度小于该值时视为移动端。
  • 按钮文本：仅在 drawer 策略下有效，默认 打开表格。

不写 mobile(...) 时，依然会有默认行为，相当于 mobile(asis, 800px, 打开表格)。

策略说明

asis在移动端不移动表格、不加类名，仅移除触发行；在桌面端正常执行移动。屏幕宽度变化时会自动切换。
enforce无论当前屏幕宽度，都执行移动。
drawer在移动端将表格放入侧滑抽屉，原位置保留一个打开按钮；在桌面端正常执行移动。屏幕宽度变化时会自动切换。

示例

au-table-float mobile(asis, 800px)移动端宽度小于 800px 时不移动
au-table-float mobile(enforce, 800px)无论宽度是否小于 800px，都移动
au-table-float left mobile(enforce)向左浮动，且始终移动
au-table-float mobile(drawer, 600px)宽度小于 600px 时使用侧滑抽屉
au-table-float mobile(drawer, 600px, 查看详情)抽屉按钮显示“查看详情”
au-table-float mobile(drawer,,打开)省略阈值（默认 800px），按钮显示“打开”

CSS 样式参数

可在触发词后追加以下 CSS 样式参数，直接赋值到表格的 style 属性上。参数不区分大小写，可与其他参数混用。

  • font-size(xxx)：设置表格字体大小，例如 font-size(14px)
  • max-width(xxx)：设置表格最大宽度，例如 max-width(300px)
  • min-width(xxx)：设置表格最小宽度，例如 min-width(100px)
  • width(xxx)：设置表格宽度，例如 width(50%)

示例

au-table-float font-size(14px)表格字体大小设为 14px
au-table-float left width(200px)向左浮动，表格宽度 200px
au-table-float mobile(drawer,600px) max-width(100%)移动端抽屉，最大宽度 100%
au-table-float FONT-SIZE(16px) Min-Width(120px)参数名大小写不敏感
注意：空值（如 font-size()）会被忽略，不会设置到表格样式上。

目标定位参数 target(...)

默认情况下，浮动后的表格会移动到全文开头。如果希望把表格放到某个段落标题附近，可以使用 target(...) 参数。

语法：target(关键词, 位置)

  • 关键词：必填。插件会搜索页面内所有 <h1> 段落标题，找到第一个标题文本中包含该关键词的标题。
  • 位置：可选，before（标题之前）或 after（标题之后）。省略时默认为 before。

示例

au-table-float target(参考资料)将表格移动到第一个标题文本包含“参考资料”的段落标题之前
au-table-float target(参考资料, before)同上，显式指定 before
au-table-float target(参考资料, after)将表格移动到第一个标题文本包含“参考资料”的段落标题之后
au-table-float left target(章节一)向左浮动，并定位到标题文本包含“章节一”的段落标题之前
au-table-float target(章节一) mobile(drawer,600px)桌面端定位到段落标题之前，移动端使用抽屉
注意：使用 target(...) 后，桌面端（及 enforce 策略）表格将不再移动到全文开头，而是直接插入到目标段落标题之前或之后；移动端 drawer 策略仍然会在表格原始位置生成打开按钮。

响应式切换

插件会监听窗口大小变化。当屏幕宽度跨越阈值时：

  • asis 策略：从桌面端变窄到移动端时，表格从 .paras 回到原位置；从移动端变宽到桌面端时，表格移动到 .paras 内。
  • drawer 策略：从桌面端变窄到移动端时，表格进入侧滑抽屉并生成打开按钮；从移动端变宽到桌面端时，抽屉和按钮被移除，表格移动到 .paras 内。

侧滑抽屉（drawer 策略）

当 drawer 策略在移动端生效时：

  • 在原表格位置插入一个 <button>，样式为 display: block; margin: 10px auto;。
  • 页面中新增两层元素：
    • 背景层：黑色半透明（opacity: 0.4），撑满整个视口，点击可关闭抽屉。
    • 主体层：白色背景，上下撑满屏幕高度，固定在左侧或右侧。默认从右侧滑出，若指定 left 方向则从左侧滑出。
  • 表格被移动到抽屉主体层内部。
  • 打开抽屉时，背景层在 0.3 秒内从透明渐变为半透明，主体层在 0.3 秒内从屏幕外滑入。

效果

  • 触发词所在的整行会被移除。
  • 在桌面端（或 enforce 策略下，或 drawer 策略下），触发词所在的 <table> 元素会被设置浮动方向，并添加类名 au-floated-table；未指定 target(...) 时，表格会被移动到全文开头（当前页面第一个 .wikiView 元素内的第一个 .paras 元素内部的开头位置），.paras 元素会添加类名 au-floated-paras；指定 target(...) 时，表格会被移动到目标段落标题之前或之后。
  • 在移动端使用 drawer 策略时，表格会被放入侧滑抽屉，原位置保留一个按钮，点击按钮打开抽屉。
  • 表格的 style 属性会根据方向设置外边距（桌面端或 enforce 策略下）：
    • 向右浮动：margin-left: 10px、margin-bottom: 10px
    • 向左浮动：margin-right: 10px、margin-bottom: 10px
  • 插件运行时会自动向 <head> 末尾插入 <style id="au-table-float-styles">，为 .au-floated-paras 内的元素提供默认样式；多次运行不会重复插入。

注入样式

本插件启动时，会略微改变页面原有行为，水平方向滚动条将不会再出现，以下是将会注入的样式：

<style id="au-table-float-styles">
        .wikiView .au-floated-paras .para,
        .wikiView .au-floated-paras .para .indent {
            overflow-x: initial;
        }
        .wikiView .au-floated-paras .para .quote {
            overflow: hidden; /*确保创建BFC，避免quote占满整行（它需要被压缩）*/
        }
        .indent.indentFolded {
            display: none !important
        }
        .au-table-float-drawer {
            padding: 6px;
        }
        .au-table-float-drawer table {
            min-width: 100%;
            font-size: 14px
        }
</style>

错误处理

如果桌面端（或 enforce 策略下）页面中不存在 .wikiView .paras 元素，表格仍会被移除触发行并设置浮动样式，同时会在表格后方显示红色错误文本：找不到.paras元素。

注意事项

  • 触发词应位于单元格内容开头，后跟可选参数。
  • 如果一行中有多个单元格包含触发词，该行也只会被移除一次。
  • 如果同一个表格中有多个行包含触发词，这些行都会被移除。
  • 插件按表格独立处理，多个表格互不干扰。
  • 只使用页面中第一个 .wikiView 元素内的第一个 .paras 元素。
  • 触发词区分大小写，必须使用 au-table-float；方向参数和策略参数不区分大小写。
