# 问题修复清单

本文档记录 AI 创作建议功能在实现阶段发现的缺陷及修复建议，按优先级排序。

## 严重（阻塞使用）

### 1. 流式对话认证失败：EventSource 无法携带 JWT 头 ✅ 已修复

- **位置**：`FCloud3.AppFront/FCloud3Front/src/components/Ai/AiChatPanel.vue`
- **问题**：前端使用原生 `EventSource` 请求 `/api/AiChat/GetSuggestions`，但项目认证方式为 `Authorization: Bearer <jwt>`（见 `httpClient.ts`）。`EventSource` 不支持自定义 HTTP 头，请求会被 `[Authorize]` 拦截为 401，AI 对话无法使用。
- **修复方式**：采用方案 A，改用 `fetch` + `ReadableStream` 消费流式响应，并在请求头中携带 `Authorization: Bearer <token>`。原 `EventSource` 代码已替换。

### 2. 后端未按 SSE 格式输出流式响应 ✅ 已修复

- **位置**：`FCloud3.App/Controllers/Ai/AiChatController.cs`
- **问题**：Controller 返回 `IAsyncEnumerable<AiChatChunk>`，ASP.NET Core 默认会将其序列化为普通 JSON 数组，而不是 `data: ...\n\n` 格式的 Server-Sent Events。即使认证问题解决了，前端 EventSource 也无法解析。
- **修复方式**：配合前端方案 A，后端改为返回 `application/x-ndjson` 流式响应。`GetSuggestions` 现直接写入 `Response.Body`，每个 `AiChatChunk` 序列化为 JSON 后加换行输出，并立即 `Flush`。

### 3. 当前词条编辑上下文未注入 ✅ 已修复

- **位置**：`FCloud3.Services/Ai/AiChatService.cs`
- **问题**：`GetWikiPathNameById` 是 TODO 存根，永远返回 `null`。前端传入 `currentWikiItemId` 后，AI 不会拿到当前词条内容。
- **修复方式**：在 `AiChatService` 中注入 `WikiItemRepo`，通过 `wikiItemRepo.GetById(wikiItemId)` 查询词条，并返回其 `UrlPathName`。`GetWikiPathNameById` 不再使用 `static` 修饰，以便访问实例字段。

### 4. 团体用量统计只统计单个 AI 实例 ✅ 已修复

- **位置**：`FCloud3.Services/Ai/AiUsageService.cs`
- **问题**：`GetGroupUsage` / `GetGroupUsageRanking` 使用 `configRepo.GetByGroupId(groupId)` 只取一个 `AiInstanceConfig`，但当前设计允许一个团体拥有多个 AI 实例。团体有多个实例时，用量会被少算。
- **修复方式**：
  - 在 `AiInstanceConfigRepo` 中新增 `GetByGroupIdAll(int groupId)`，返回某团体的所有 AI 实例配置。
  - `AiUsageService.GetGroupUsage` 改为汇总该团体下所有实例的 `TotalTokens`。
  - `AiUsageService.GetGroupUsageRanking` 改为跨所有实例按 `UserId` 聚合总量与调用次数。
  - 在 `AiUsageServiceTest` 中新增 Group 300 含两个 AI 实例的测试数据，并补充对应测试用例。

### 5. 前端“获取模型列表”返回格式与后端不一致 ✅ 已修复

- **位置**：`FCloud3.App/Controllers/Identities/AiInstanceConfigController.cs`
- **问题**：后端返回 `ApiResp(models)`，即 `data` 是 `string[]`；前端模型 `AiAvailableModelsResult` 期望 `data.Models`。`res.Models` 为 `undefined`，模型下拉框不会填充。
- **修复方式**：后端改为返回 `ApiResp(new { Models = models })`，使 `data` 结构与前端 `AiAvailableModelsResult` 一致。

## 重要（影响正确性）

### 6. `SaveMessages` 用 `history.Count` 作为 `Order` 基准，可能导致重复 ✅ 已修复

- **位置**：`FCloud3.Services/Ai/AiChatService.cs`
- **问题**：当 `MaxContextMessages` 截断历史时，`history.Count` 小于数据库中的实际消息数，`Order` 会从中间值开始，导致后续消息 `Order` 重复。
- **修复方式**：
  - `SaveMessages` 移除 `existingCount` 参数，改为通过 `messageRepo.GetMaxOrder(conversationId)` 获取当前最大 `Order`。
  - 新消息的 `Order` 基于数据库实际最大值，不再受上下文截断影响。
  - 更新 `MessageCount` 时改用动态计数：在每次 `TryAddMessage` 成功后递增 `addedCount`，最终 `conv.MessageCount += addedCount`，避免硬编码 `+= 2`。

### 7. 月限额未生效 ✅ 已修复

- **位置**：`FCloud3.Services/Ai/AiChatService.cs`
- **问题**：实体 `AiInstanceConfig` 有 `MonthlyTokenLimit`，但 `GetSuggestions` 只检查了 `DailyTokenLimit`。
- **修复方式**：在 `DailyTokenLimit` 检查之后，增加 `MonthlyTokenLimit` 检查。计算本月起始日期（`new DateTime(DateTime.Now.Year, DateTime.Now.Month, 1)`），通过 `usageService.GetConfigUsage(config.Id, monthStart)` 查询本月用量，若达到限额则返回提示并终止。

### 8. 词条内容不应直接返回 HTML，应提供原始结构化文本给 AI

- **位置**：`FCloud3.Services/Ai/AiChatService.cs` 调用 `AiToolService.GetWikiContent` 处
- **问题**：当前 `GetWikiContent` 直接返回 `WikiParsingResultItem.Content`（HTML）。HTML 标签占用 token、包含样式/脚本等噪声，且如果后续 strip 成纯文本，会丢失标题、列表、表格等层级结构。
- **修复建议**：改为返回接近原始源码的结构化文本，而不是解析后的 HTML：
  - **文本段落**：直接读取对应 `TextSection.Content`（解析前的 Markdown/Wiki 源码），保留 `##` 标题、列表、加粗等层级标记。少量专有 Wiki 语法暂时不处理。
  - **表格段落**：读取 `FreeTable` 的原始数据，转换为 CSV 格式。
  - **文件段落**：仅提供段落名；如果段落名为空，则 fallback 到文件名。
  - 拼接时按段落顺序组织，用 `## {段落名}` 分隔不同段落。

### 9. `AiToolService.SearchWiki` 参数未使用

- **位置**：`FCloud3.Services/Ai/AiToolService.cs:11`
- **问题**：构造函数注入 `quickSearchService` 但从未使用，实际搜索用的是 `wikiItemRepo.QuickSearch`。编译会产生 warning CS9113。
- **修复建议**：要么使用 `quickSearchService` 实现更准确的搜索，要么移除未使用的依赖。

## 一般（体验或规范）

### 10. AI 配置修改权限比方案更严格

- **位置**：`FCloud3.Services/Identities/AiInstanceConfigService.cs:117-120`
- **问题**：方案要求“团体所有者/管理员可设置”，代码只检查 `group.OwnerUserId != _userId`，排除了管理员。
- **修复建议**：确认业务需求。若需管理员也能设置，增加对管理员身份的判断。

### 11. 前端对话标题未动态更新

- **位置**：`AiChatPanel.vue` / `AiChatMessageArea.vue`
- **问题**：`AiChatPanel` 的 `currentTitle` 固定为 `'AI 创作助手'`，`AiChatMessageArea` 的 `title` 固定为 `'对话'`。重命名或切换对话后，界面标题不会变化。
- **修复建议**：在加载对话列表/消息时，同步设置当前对话标题。

### 12. `AiInstanceConfigController.Set` 权限验证方式不直观

- **位置**：`FCloud3.App/Controllers/Identities/AiInstanceConfigController.cs:85-86`
- **问题**：通过 `IAuthGrantableRequestModel`（`AuthGrantOnId => GroupId`）加 `[UserTypeRestricted]` 间接完成授权，没有显式 `[AuthGranted]`。虽然 `AuthGrantActionFilter` 能识别，但可读性较差。
- **修复建议**：显式加上 `[AuthGranted(formKey: nameof(model.GroupId))]`，让权限意图更清晰。

## 设计偏离说明

当前实现与原始方案文档相比有以下重要调整，后续文档应同步更新：

- **“团体”扩展为“AI 实例”**：一个团体可拥有多个 AI 实例，对话和用量记录使用 `AiInstanceConfigId` 而非 `GroupId` 关联。
- **`AiInstanceConfig` 实体位置**：实际放在 `FCloud3.Entities/Ai/`，而非方案中的 `FCloud3.Entities/Identities/`。
- **`AiConversation` 关联字段**：使用 `CurrentWikiItemId` 替代 `CurrentWikiPathName`。
- **前端未嵌入词条编辑页**：目前只有独立页面 `/AiChat/:aiInstanceConfigId?`，尚未在 `WikiItemContentEdit.vue` 或 `ViewWiki.vue` 中集成 AI 面板。
